Plan: Миграция имён файлов в camelCase (инкрементально)

Кратко
Привести имена файлов и папок в `src` к единой конвенции: camelCase для файлов/папок (модули), PascalCase — только для имен классов внутри файлов. Миграция будет инкрементальной: переименовываем по батчам (папкам), после каждого батча обновляем импорты, прогоняем сборку и тесты.

Порядок и батчи (первичный план)
1) entities — доменные сущности (высокий приоритет, много ссылок).
2) utils — кросс-катционные вспомогательные модули.
3) presentation/bot — CommandsRegistry и handlers.
4) domain/services — сервисный слой.
5) qBittorrent — интеграционные модули.
6) searchEngines — поисковые движки.
7) misc: корневые конфиги и миграции (в конце).

Общая процедура для каждого батча
- Создать ветку: `git checkout -b rename/camelcase-<batch>`
- Подготовить `rename-map.csv` для батча (старый путь,новый путь)
- Выполнить переименование: для обычных файлов `git mv <old> <new>`; для case-only на macOS — временный шаг `git mv Old.ts Old.tmp && git mv Old.tmp old.ts`.
- Запустить скрипт обновления импортов (локально): `node scripts/update-imports.js rename-map.csv` (скрипт заменяет только часть пути между кавычками, не меняет расширения).
- Запустить сборку TypeScript: `npx tsc -p tsconfig.json` или `npm run build`.
- Запустить тесты: `npx vitest` (или `npm test`). Исправить ошибки по ходу.
- Коммитить логичные группы: `git add -A && git commit -m "chore: rename <batch> to camelCase"`.
- Создать PR/или push и дождаться CI зеленого результата.

Особые замечания
- Migrations: по возможности не переименовывать файлы миграций — если переименование обязательно, обновите все ссылки в коде и инструменты миграций. Переименование миграций делает их имена отличными в системе версий и может нарушить порядок-пользования в некоторых инструментах.
- macOS case-only: в моём аудите не обнаружено конфликтов, но всё равно использовать temp-rename шаблон при подозрениях.
- Сохранить расширения в import-спецификаторах: если проект использует явное `.js` в импортах, скрипт обновлений должен изменять только базовое имя.
